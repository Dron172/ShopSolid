Пояснения по примененным принципам SOLID:
1. Избавление от магических чисел, 
получение списка продуктов и сохранение в массиве
2.DRY - добавление аналогичных товаров в одну строку
3.Open-closed principle - расширение функциональности класса за счет интерфейса
4. Dependency inversion principle - принцип закрытости изменения не вносятся в код, используемый классом.
5. Liskov substitution principle - срочный заказ может заменить обычный заказ.
